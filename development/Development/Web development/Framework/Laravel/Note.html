Flow

  1. Install laravel

  2. Create laravel project 

  3. Create database 

  4. Configure database 

  5. Create migration file for table 

  6. Create seed for table data ( optional ) 

  7. Create view for presentation

  8. Create controller to handle route request 

  9. Create route to redirect http request to controller 

  10. Create model for the table to handle the data configuration and interact with data 

  11. Create middleware to handle the access and authentication 

  12. Create test case

Basic

 Term Description
 MVC
  • MVC stands for Model View Controller
  • Model used to handle data and business logic
  • View used to present data to user 
  • Controller used to receive user request and call appropriate resource
 Model
  •  In laravel, each table has a corresponding Model that allow us to interact with table.
  • It used to retrieve, insert, delete and update information to data table 
  • It can used to set relationship between the table 
 Controller
  •  It directing traffic between views and models.
 View
  •  Used to display data to user 
 Middleware
  • Middleware acts as middle man between request and response.
  • Middleware is filtering mechanism that filtering incoming HTTP request before it routed to controller
  • Location : app/Http/Middleware 
  • Example use : 
    -  Verify user is authenticated or not then redirect user to specific page 
    -   To authenticate the action based on role 
 Blade
  •  Is a template engine for laravel framework 
 Route
  • It perform mapping to url request to a specific controller action.
  • Location : app/Http/routes.php 
  • Example : http://localhost/route
 Eloquent ORM
  •  ORM stand for Object Relation Mapping 
  • It used to interact with database ( CRUD , create , read , update , delete )
