Migrate problem 

  • Migrate error - cannot create table problem 

 SQLSTATE[42000]: Syntax error or access violation: 1071 Specified key was too long
 Solution : 
  • Open \app\Providers\AppServiceProvider.php
  •  Add this after: use Illuminate\...   :
    use Schema;
  • Add this line in function boot() :
    \Schema::defaultStringLength(191); 

  • Migrate error - row problem

 columns "created_at" and "updated_at" are not found in this table.
 Solution :
  • Create model for that table 
    php artisan make:model <table_model_name>
  • Open the new created model file and add :  [ add App/<model_name>.php ] 
    public $timestamps = false


  • Migrate error - cannot add foreign key constraint  

 Solution : 
  • Create "Parent" table and migrate first then only create "Child" table and perform migrate 

  • Add   
    $table->unsignedInteger('user_id') ->unsigned()->index()->nullable();
  • Add another schema 
     Schema::table('table_name', function($table) {
              $table->foreign('user_id')->references('id')->on('users')->onDelete('set null');
      });

    • MethodNotAllowedHttpException

    Solution : Check your request method is correct or not 


    • No 'Access-Control-Allow-Origin' header when call AJAX to localhost 


    • Step 1 : Create Middleware to handle access origin header 
      php artisan make:middleware Cors

    • Step 2 : Add following code in app/Http/Middleware/Cors.php
      public function handle($request, Closure $next)
{
	return $next($request)
		->header('Access-Control-Allow-Origin', '*')
		->header('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE, OPTIONS')
		->header('Access-Control-Max-Age', '1000')
		->header('Access-Control-Allow-Headers', 'Content-Type, Authorization, X-Requested-With');		
}

    ** You can replace * to any ip address 

    • Step 3 : Load middleware by add following line in app/Http/Kernel.php $routedMiddleware function 
      'cors' => \App\Http\Middleware\Cors::class,

    • Step 4 : Register public middleware by add following line in app/Http/Kernel.php $middleware function  ( If still not working ) 
      'cors' => \App\Http\Middleware\Cors::class,

    • Step 5 : Use middleware on the API route by add following line in app/Providers/RouteServiceProvider.php 
      protected function mapApiRoutes()
{
    Route::prefix('api')
         ->middleware('api')
         ->middleware('cors')
         ->namespace($this->namespace)
         ->group(base_path('routes/api.php'));
}
